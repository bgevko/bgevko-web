## Fundamentals
- `pwd`: Print working directory.
- `ls`: List directory contents.
- `cd <directory>`: Change directory.
- `touch <filename>`: Create an empty file.
- `mkdir <directory>`: Create a directory.
- `rm <filename>`: Remove a file.
- `rm *.<extension>`: Removes all files with specified extension
- `rm -r <directory>`: Remove a directory recursively.
- `mv <source> <destination>`: Move or rename files/directories.
- `mv ^exception <destination>`: Move everything but the exception into the destination
- `rm -rf *(D)`: Remove (Delete) all files in directory, including hidden files
- `cp <source> <destination>`: Copy files to destination
- `cp -rp <dir> <destionation>` Copy dir and preserve permissions (and other attributes) to destination
- `chmod +x exetutable.sh`: Enables a file to be executed.

## User Management
- `whoami`: Print the current user name
- `id`: Print real and effective user and group IDs
- `useradd <username>`: Add a new user.
- `userdel <username>`: Delete a user.
- `passwd <username>`: Set or change password for a user.
- `id <username>`: Display user and group ID.
- `groups <username>`: Show groups a user belongs to.
- `sudo <command>`: Run a command as another user (default is root).
- `su - <username>`: Switch user.

## Shutting Down and Rebooting
- `shutdown -h now`: Immediately shut down the system
- `shutdown -r now`: Immediately restart the system
- `shutdown -h +m`: Shut down the system in `m` minutes
- `reboot`: Restart the system (equivalent to `shutdown -r now`)
- `poweroff`: Power off the system (equivalent to `shutdown -h now`)
- `halt`: Halt the system

## Systemctl Commands
- `systemctl start <service>`: Start a service.
- `systemctl stop <service>`: Stop a service.
- `systemctl restart <service>`: Restart a service.
- `systemctl reload <service>`: Reload service configuration without stopping.
- `systemctl status <service>`: Check the status of a service.
- `systemctl enable <service>`: Enable a service to start at boot.
- `systemctl disable <service>`: Disable a service from starting at boot.

## Networking
- `ip addr`: Show IP addresses of all network interfaces
- `ip route`: Show system's routing table
- `netstat`: Show network connections, routing tables, interface statistics, etc.
- `ss`: Utility to investigate sockets
- `ping <host>`: Send ICMP ECHO_REQUEST to network hosts
- `traceroute <host>`: Show the route packets take to a network host

## SSH
- `ssh <username>@<host>`: Connect to a host as a user.
- `ssh-keygen`: Generate an SSH key pair.
- `ssh-copy-id <username>@<host>`: Copy public SSH key to a host for passwordless login.
- `scp <source> <username>@<host>:<destination>`: Secure copy file/directory to a remote host.
- `systemctl restart sshd`: Restart the SSH service.
- `systemctl status sshd`: Check the status of the SSH service.
- `~/.ssh/id_rsa`: Default private SSH key location.
- `~/.ssh/id_rsa.pub`: Default public SSH key location.
- `~/.ssh/authorized_keys`: File containing public keys allowed to connect.
- `eval "$(ssh-agent -s)"`: Start the SSH agent in the background.
- `ssh-add ~/.ssh/id_rsa`: Add your private key to the SSH agent.
- `ssh-add -l`: List keys added to SSH agent.
- `ssh-add -D`: Remove all keys from the SSH agent.

## Updates and Package Management
- `sudo apt update`: Update the package list
- `sudo apt upgrade`: Upgrade all upgradable packages
- `sudo apt dist-upgrade`: Upgrade packages and handle dependencies
- `sudo apt autoremove`: Remove unnecessary packages
- `sudo apt install <package-name>`: Install a package
- `sudo apt remove <package-name>`: Remove a package
